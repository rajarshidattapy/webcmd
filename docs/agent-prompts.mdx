# Prompt Cookbook

Describe the target, outcome, required fields, authentication context, and safety boundaries. The agent handles discovery, browser work, implementation, and verification.

## Find and Use an Existing Adapter

```text
Use `webcmd-usage` to find an existing adapter for Hacker News discussions about browser automation. Return title, URL, author, points, and source for the best results. This is a public, read-only task: do not post, vote, save, or create a new adapter unless no existing command can complete it.
```

## Complete a One-Off Browser Task

```text
Complete this read-only task on https://example.com/jobs: find product-design roles in London and return title, company, location, salary_text, posted_at, and URL. The site is unfamiliar, so use the right browser workflow after checking `webcmd-usage`; do not apply, create an account, or change saved searches.
```

## Create a Reusable CLI

```text
Create a private Webcmd CLI for this supplier portal. Given a part number, return price, stock, MOQ, lead_time, and product_url as stable JSON-friendly rows. The portal is public; keep it read-only, reuse existing adapters or helpers where possible, and verify the command before finishing.
```

## Use a Logged-In Website

```text
Use my logged-in `work` browser profile to read invoices from the Acme billing dashboard. Return invoice_id, customer, due_date, amount, status, and detail_url; pause if I need to sign in, do not store credentials in code, and do not change any invoice or account data.
```

## Extend an Existing CLI

```text
Extend the existing Webcmd GitHub adapter with a read-only command that lists open pull requests assigned to me. Return number, title, repository, author, updated_at, and URL as stable JSON; use my existing GitHub authentication only if needed, reuse existing helpers, and verify the command without modifying any pull request.
```

## Repair a Broken CLI

```text
The command `webcmd example jobs "designer"` now returns empty rows. Diagnose and repair it, preserving title, company, location, salary_text, posted_at, and URL in its output schema. Treat it as read-only, do not store credentials or alter job listings, and run verification before explaining the fix.
```

## Create a Plugin

```text
Create a private `webcmd-acme` plugin for our internal tools: Jira release notes should return version, title, date, and URL; Confluence lookup should return title, excerpt, owner, and URL; and Acme invoices should return invoice_id, customer, amount, status, and detail_url. Use my `work` profile only where authentication is needed, keep all workflows read-only, include install instructions, and verify the plugin.
```

## Publish a Community Plugin

```text
Prepare my `webcmd-acme` plugin for contribution to the AgentR Webcmd repository. Place it under `plugins/acme/`, make the manifest name match the directory, include version, description, Webcmd compatibility, and my author name and GitHub handle, improve the README and examples, run the repository plugin checks, and open a draft pull request returning its URL, workflow summary, and verification evidence. Use my authenticated GitHub account only for that draft PR; do not merge, publish a package, or modify unrelated repository files.
```

## Ask for a Plan First

```text
Before writing code for the Acme billing dashboard, inspect existing Webcmd adapter patterns and propose the smallest private, read-only design. Include command names, required invoice_id, customer, due_date, amount, status, and detail_url fields, `work` profile authentication needs, safety boundaries, and a verification plan; do not create or modify any adapter until I approve the plan.
```
