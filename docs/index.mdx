# Webcmd

Webcmd is an agent runtime for creating CLIs out of websites, apps, APIs, browser sessions, and local tools.

The user gives an agent a goal. The agent uses Webcmd to crawl the target, understand the UI and browser state, discover useful requests or workflows, and turn that knowledge into repeatable commands.

## Why It Exists

Most useful software does not expose the exact CLI an agent needs. It might be a website, a logged-in dashboard, a desktop app, a JSON endpoint hidden behind browser state, or a local binary with awkward output.

Webcmd gives agents a standard way to:

- Explore those surfaces safely.
- Create deterministic commands from repeated workflows.
- Reuse existing commands before crawling again.
- Package commands into plugins.
- Heal commands when sites change.

## Who Uses Webcmd

Humans do not need to inspect network requests, write selectors, or manually drive browser sessions.

Humans use Webcmd by prompting an agent:

```text
Create a Webcmd CLI for this supplier portal. Given a part number, return price, stock, MOQ, lead time, and product URL as JSON. Keep it private and verify it.
```

The agent does the implementation work and reports the final command.

## Core Idea

Agents should not repeat fragile browser work forever.

Webcmd turns successful exploration into a command:

```text
Prompt -> agent explores -> adapter is created -> command is verified -> future runs use the CLI
```

The goal is a stable command surface that humans and agents can both trust.
