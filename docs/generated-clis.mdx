# Generated CLIs

The output of a successful Webcmd authoring task is a reusable CLI command.

Agents should create commands that future agents can discover and call without repeating the original crawl.

## Registry

```bash
webcmd list
webcmd list -f json
```

The registry includes built-in adapters, private adapters, plugin adapters, and registered external CLIs.

## Where Commands Live

| Source | Location | Use for |
| --- | --- | --- |
| Built-in | `clis/<site>/<command>.js` | Commands shipped with Webcmd. |
| Private | `~/.webcmd/clis/<site>/<command>.js` | Personal or company-local workflows. |
| Plugin | Plugin package directory | Shareable adapter packs. |
| External | `~/.webcmd/external-clis.yaml` | Existing local binaries exposed through Webcmd. |

## Command Contract

A generated command should have:

- A clear name, such as `webcmd acme invoices`.
- Arguments and options that match the user's workflow.
- Stable JSON output.
- Human-readable help text.
- A declared strategy.
- Verification or smoke coverage.

Prompt example:

```text
Create `webcmd acme invoices`. It should accept `--status` and `--since`, return stable JSON rows, and include help text that explains required login profile.
```

## Private Adapters

Private adapters are the default for user-specific workflows. They are good for dashboards, internal tools, and experiments.

Prompt example:

```text
Keep this adapter private under my Webcmd user directory. Do not modify the upstream repository.
```

## Local Overrides

An agent can copy a built-in adapter into the user directory and modify it locally.

Prompt example:

```text
Patch the Twitter adapter locally for my workflow. Keep it as a local override and explain how to reset it later.
```

Useful management commands:

```bash
webcmd adapter status
webcmd adapter eject <site>
webcmd adapter reset <site>
```

User adapters override built-in adapters. Plugins can override both.

## Plugins

When several commands belong together, ask the agent to package them as a plugin.

```text
Move these Acme adapters into a Webcmd plugin. Include a README, examples, and plugin install instructions.
```

## External CLIs

External CLIs are useful when the command already exists as a local binary. Webcmd makes it discoverable to agents through the same registry.

```text
Register `releasectl` as a Webcmd external CLI. Add the install command, description, and verify that agents can discover it with `webcmd list -f json`.
```

## Updating Generated CLIs

Ask agents to preserve compatibility unless you explicitly want a breaking change.

```text
Update `webcmd acme invoices` to include currency and paid_at. Keep all existing columns stable and add tests or smoke verification.
```

## Healing Generated CLIs

When a site changes, the agent should repair the command rather than create a duplicate.

```text
Heal `webcmd acme invoices`. It now returns empty results. Preserve the command name and output schema, repair the adapter, and report what changed.
```

This keeps downstream prompts and automations stable.
