Markdown

Standard file extension.md

Markdown is a lightweight markup language for creating formatted text using a plain-text editor. John Gruber created Markdown in 2004 with Aaron Swartz as a collaborator on the syntax. The language is commonly used for readme files, documentation, notes, and messages where readable source text matters.

The central design goal of Markdown is readability. A Markdown document should be publishable as plain text without looking like it has been marked up with tags or formatting instructions. That goal helped Markdown become a common interchange format for developer documentation and publishing systems.

Markdown processors convert the source text into HTML or other output formats. Implementations often add extensions for tables, task lists, fenced code blocks, footnotes, and automatic links. These extensions are widely used, but the exact behavior can vary between renderers.

History

John Gruber released the original Markdown syntax description and a Perl converter in 2004. The format spread because the input stayed easy to read in email, source control, issue trackers, static site generators, and documentation systems.

Examples

A heading begins with number signs, a list item begins with a hyphen or number, and inline links use square brackets followed by a URL in parentheses. Code examples may be indented or placed inside fenced code blocks.
